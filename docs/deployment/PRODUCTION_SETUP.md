# Enterprise Production Setup Guide
## Blockchain-Enabled Human Organ Transplantation & Smart Organ Transport Platform

This guide describes how to configure and deploy the platform in a production cloud environment.

---

## 1. Production Security Hardening
Before running the deployment script, ensure you apply the following security policies:
1.  **SSL/TLS Certificates**: Configure Nginx with valid domain certificates (Let's Encrypt or similar).
2.  **Firewall Settings**: Restrict port access. Only ports `80` and `443` should be accessible from public networks.
3.  **Secrets Management**: Load all variables using Docker Secrets or vault management systems. Do not save credentials in cleartext files.
4.  **Database Access**: Ensure MongoDB replica set ports (`27017`) are blocked by network firewalls.

---

## 2. Infrastructure Deployment

### Step 1: Deploy MongoDB Replica Set
Deploy three MongoDB nodes across isolated availability zones to ensure high availability:
```bash
docker stack deploy -c docker-compose.mongo-replica.yml prod-db
```

### Step 2: Initialize Hyperledger Fabric Network
Generate cryptographic certificates and register peer identities for each hospital organization using Fabric-CA:
```bash
cd blockchain/scripts
./bootstrap-network.sh --profile production
```

### Step 3: Deploy Frontend & Backend Services
Start the application services behind Nginx load balancers:
```bash
docker stack deploy -c docker-compose.prod-app.yml prod-services
```

---

## 3. Registering Physical ESP32 Devices
1.  Log in as a System Administrator.
2.  Navigate to the **Devices Configuration** panel.
3.  Add the new device's UUID and upload its public key hash.
4.  Flash the ESP32 hardware using the config binary generated by the portal.
